Javascript - creating a quiz part 1

There are many different ways of creating a quiz using JavaScript. As always with programming the first step should be to identify the problems you need to solve. Fundamentally they are as follows:

  1. Create a way for users to input answers to questions
  2. Create a way to test those answers against the correct ones
  3. Create a way to store the number of right answers
  4. Create a way to communicate the results to the user. This itself could take a number of forms: answer-by-answer, or only at the end, or both. It might also involve providing the correct answers, or just providing a score.

We’ll tackle each problem one by one. This is a good way to tackle programming questions, as you can search for sample code that solves each problem.

In this part we’ll look at how to create a way for users to input information (answers to questions).

Challenge 1. Create a way for users to input answers to questions

There are two basic ways to take inputs from users:

  • A prompt window
  • A form

Option 1: The prompt window

A prompt window is a window which pops up on the user’s computer. They are more commonly referred to as ‘pop up windows’.

A prompt window specifically asks for something, however - such as a password, or name. This information is then normally used by the script to do something (take them to another page, or generate a greeting, for example).

If you want to create a prompt window you use something in JavaScript called the prompt method.

This is normally combined with creating a variable to store whatever the user enters in the prompt window.

Here, for example, is some typical JavaScript which does just that:

var userChoice = prompt('What is your name?');

The code above has two parts: to the left of the equals operator, the first part - var userChoice = - creates a variable called userChoice.

The second part, to the right of the equals operator, generates a prompt window for the user to enter something.

The equals operator assigns the value entered in the prompt window to the variable, so it can be used elsewhere in the script.

To create the prompt window the script uses the prompt method. The prompt method has one parameter: the message that the window contains: 'What is your name?'

When the script runs on the HTML page, a popup window will appear prompting the user to enter some text or numbers. Whatever they enter will be stored in that variable (userChoice).

You could do all sorts of things with that variable: test it against correct answers or a correct password, use it in calculations, use it to generate greetings (such as ‘Hello Paul!’), and so on.

This tutorial, from the online book JavaScript and jQuery, uses a prompt window. The tutorials on Codecademy tend to use prompt windows as well.

The problem with the prompt window, however, is that it’s not the most pleasant way to conduct a quiz. Popup alerts are normally associated with computer problems and annoying adverts.

For that reason, although I’ve explained what the prompt method is (because you will most likely come across tutorials which use it) we are not going to use the prompt method. We need to find a better way.

Option 2: The form

Unlike a prompt window, a form sits within the webpage itself. It is also written in HTML, not JavaScript (the JavaScript then interacts with the form in the same way that it can interact with other HTML elements).

Dive Into HTML5 has a chapter on forms which provides a useful introduction to coding forms in HTML. Take some time to go through that…

OK, done? Here are some key points to remember:

  • You create a HTML form by using the <form> tag.
  • Within the <form> and </form> tags you will need a way for the user to input information, and a way for them to submit it.
  • Information can be inputted using text fields, radio buttons or checkboxes, drop-down lists, sliders and other options.
  • They are normally submitted with a button.
  • All the ways of inputting and submitting are created using the <input> tag.
  • But what type of input (box, button, etc.) is decided by the associated type= attribute. <input type="text">, for example, specifies a text field.
  • Any inputs which are used to store information (i.e. all of them apart from the ‘submit’ button) need a name= attribute.

There are many more ways for people to input information, including:

  • <input type="checkbox">
  • <input type="email">
  • <input type="radio">
  • <input type="range">

You can find a list in the ‘Attribute values’ table on this page

The chapter on forms from *Dive Into HTML5 explains how these different types work on browsers and mobile phones. For example, the “date” type creates a small calendar; “time” creates buttons for you to increase and decrease times (“number” can do the same). Specifying “email” or “url” determines the keyboard that iPhone brings up to help you type. You can even create a colour picker. Also see this overview on HTML5 Rocks*

The name attribute is like a variable: it basically assigns whatever is entered in that field to that variable. So <input name="age"> will take whatever is inputted in that field and put it in the variable age.

The submit button

Buttons to submit the information are created with the same <input> tag but the type="submit" attribute and value. For example, <input type="submit">.

You can also label the button with the value= attribute. For example <input type="submit" value="Submit"> puts ‘Submit’ on the button, while <input type="submit" value="Search"> puts ‘Search’ on the button.

Putting it all together

The HTML for a simple form which asks a question, provides a box to answer, and a button to submit, would look like this:

<form>
<h3>What is the capital of Wales?</h3>
<input type="text" name="question1">
<input type="submit" value="Submit">
</form>

If your answer was numerical you could change the type as follows:

<form>
<h3>What is 2+2?</h3>
<input type="number" name="question1">
<input type="submit" value="Submit">
</form>

And if you wanted a multiple choice question you could use the “radio” type like so. Note that you also need to label the boxes with some text:

<form>
<h3>Which of the follow is spelt correctly?</h3>
<input type="radio" name="question1" value="They cocked their hats">They cocked their hats<br />
<input type="radio" name="question1" value="They cocked they're hats">They cocked they're hats<br />
<input type="radio" name="question1" value="They cocked there hats">They cocked there hats<br />
<input type="submit" value="Submit">
</form>

You need to add some text after the HTML for the radio box so people know what they’re ticking. The “value” attribute is invisible to the user and indeed could be abbreviated in the above example to “their” in the first answer, and so on, as the value is only what you’re recording.
